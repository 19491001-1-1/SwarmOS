## 目标

实现 Agent 实时活动日志。用户点击 Agent 可以看到它的 Activity 标签页，展示 Thinking / Working / Output / Idle / Sending message 等状态流水线。

---

## 参考截图描述

Activity 页面是一个时间线列表，每行格式：
```
HH:MM:SS  ● STATUS  detail text
```
- Thinking：黄色圆点
- Working：橙色圆点
- Output：蓝色圆点（附带输出内容摘要）
- Idle：灰色圆点
- Sending message：绿色圆点（附带目标 channel）

## 协议扩展

### shared/src/protocol.ts 新增

```ts
type AgentActivity = {
  id: string;
  agentId: string;
  type: 'thinking' | 'working' | 'output' | 'idle' | 'sending' | 'error';
  detail?: string;      // 可选的补充文本
  createdAt: string;
};
```

### Daemon → Server 新增消息

```ts
| { type: 'agent:activity'; agentId: string; activityType: AgentActivity['type']; detail?: string }
```

### Browser 推送新增

```ts
| { type: 'agent:activity'; agentId: string; activity: AgentActivity }
```

## 实现任务

### Task 1：协议 & 验证

- `packages/shared/src/protocol.ts`：添加 `AgentActivity` 类型和 `agent:activity` 消息
- `packages/shared/src/validation.ts`：添加对应 Zod schema
- 测试：`packages/shared/test/protocol.test.ts` 补充 activity 消息解析测试

### Task 2：Server 存储 & API

- `packages/server/src/db.ts`：添加 `activities` 表（id, agentId, type, detail, createdAt），限制每个 agent 最多保留 500 条
- 新增 `GET /api/agents/:id/activities` 端点，返回最近 200 条，按 createdAt desc
- `daemonSocket.ts`：处理 `agent:activity` 消息，存入 DB，通过 events 推送给 browser
- 测试：`packages/server/test/agentsApi.test.ts` 补充 activities 接口测试

### Task 3：Daemon 上报 Activity

- `packages/daemon/src/agentProcessManager.ts`：在以下时机上报 activity：
  - 收到 deliverMessage → 上报 `working`（detail: "Message received"）
  - spawn 进程前 → 上报 `thinking`
  - 解析到 bridge marker → 上报 `sending`（detail: "channel:channelId"）
  - fallback reply → 上报 `output`（detail: stdout 前 100 字符）
  - 进程退出 → 上报 `idle`
  - 进程错误 → 上报 `error`
- `packages/daemon/src/daemonClient.ts`：新增 `sendActivity()` 方法

### Task 4：Web UI

- `packages/web/src/api.ts`：添加 `getAgentActivities(agentId)` 函数
- 新建 `packages/web/src/components/AgentDetailPanel.tsx`：
  - 顶部 tab 栏：PROFILE | ACTIVITY（像素风 tab，选中黄底）
  - PROFILE tab：显示现有 agent 信息（name, runtime, model, status, machineId, createdAt）
  - ACTIVITY tab：时间线列表，实时通过 WebSocket 追加新条目
  - 每行：时间 + 彩色状态方块 + 状态文字 + detail
- `packages/web/src/App.tsx`：点击 sidebar agent 名字时，右侧面板切换到 AgentDetailPanel
- WebSocket handler：处理 `agent:activity` 事件，追加到对应 agent 的 activity 列表
