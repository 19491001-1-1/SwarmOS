# v0.2.3 — Cloudflare Hub 安全与测试加固

> **给 Claude Code 的执行指令：** 实现本文档描述的所有功能。按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。循环直到所有验收条件满足。完成后运行 `pnpm verify` 确认全绿。不要停在方案设计，必须完成代码、测试、文档和提交。

---

## 目标

加固 Cloudflare 旁路 hub，使它适合长期公网运行：

1. 浏览器用户鉴权。
2. Cloudflare Worker / Durable Object 集成测试。
3. REST API 输入校验。

---

## 背景

当前 Cloudflare hub 已可用，但还有三个风险：

- `/api/*` 和 `/ws` 面向公网基本无用户鉴权。
- `packages/cloudflare` 只有 typecheck，没有 Worker/DO 行为测试。
- Worker route 中大量使用 `body as Partial<Agent>` 这种类型断言，缺少运行时校验。

v0.2.3 要补齐这些缺口。

---

## 技术选型

- 输入校验：优先复用 `zod` 和 `@crewden/shared` 中已有 schema。
- Worker 测试：使用 `@cloudflare/vitest-pool-workers`。
- 浏览器鉴权：先实现单用户 token 模式，不引入 OAuth。

---

## 鉴权设计

### Daemon auth

保持现有：

```text
/daemon/connect?key=<DAEMON_API_KEY>
```

改进：

- 使用 timing-safe compare。
- key 不允许为空。

### Browser auth

新增 Cloudflare secret：

```text
WEB_AUTH_TOKEN
```

所有 browser 访问需要携带 token：

REST:

```http
Authorization: Bearer <WEB_AUTH_TOKEN>
```

Browser WebSocket:

```text
/ws?token=<WEB_AUTH_TOKEN>
```

本地 web 通过环境变量配置：

```text
VITE_WEB_AUTH_TOKEN=<WEB_AUTH_TOKEN>
```

说明：

- v0.2.3 不实现多用户登录，不做 cookie session。
- token 模式足够保护当前单人/小团队公网部署。
- 后续多用户再引入 Cloudflare Access 或 OAuth。

## 实现任务

### Task 1：新增 shared API 输入 schema

文件：

```text
packages/shared/src/validation.ts
```

新增并导出：

```ts
CreateAgentRequestSchema
PatchAgentRequestSchema
CreateMessageRequestSchema
```

规则：

- create agent：
  - `name`: required string, min 1
  - `runtime`: required RuntimeId
  - `displayName`, `description`, `model`, `systemPrompt`, `machineId`: optional string
- patch agent：
  - `machineId`, `displayName`, `model`, `systemPrompt`: optional string
  - 至少一个字段存在
- create message：
  - `senderName`: required string, min 1
  - `content`: required string, min 1
  - `agentId`: optional string

测试：

```text
packages/shared/test/protocol.test.ts
```

覆盖 valid / invalid cases。

### Task 2：Cloudflare Worker 使用 Zod 校验

文件：

```text
packages/cloudflare/src/index.ts
```

要求：

- `POST /api/agents` 使用 `CreateAgentRequestSchema.safeParse`。
- `PATCH /api/agents/:id` 使用 `PatchAgentRequestSchema.safeParse`。
- `POST /api/channels/:id/messages` 使用 `CreateMessageRequestSchema.safeParse`。
- 解析失败返回 HTTP 400：

```json
{ "error": "Invalid request body", "issues": [...] }
```

- 不再对 request body 直接 `as Partial<Agent>` 做业务信任。

### Task 3：Server 同步使用 API schema

文件：

```text
packages/server/src/routes/agents.ts
packages/server/src/routes/messages.ts
```

要求：

- 本地 server 也使用同一套 request schema。
- 保持现有错误语义尽量兼容，但 invalid body 必须返回 400。
- 更新现有 server tests。

### Task 4：浏览器鉴权

Cloudflare Worker：

- 所有 `/api/*` route 调用前检查 browser auth。
- `/ws` 建立前检查 token。
- `/daemon/connect` 继续使用 daemon key，不使用 browser token。
- `OPTIONS` 继续允许 CORS preflight。

新增 helper：

```ts
function requireBrowserAuth(request: Request, env: Env): Response | null;
function timingSafeEqual(a: string, b: string): Promise<boolean> | boolean;
```

要求：

- 缺失 token 返回 401。
- token 错误返回 401。
- `WEB_AUTH_TOKEN` 未配置时，生产环境必须拒绝请求；本地测试可以通过 test env 配置。

Web：

- `packages/web/src/api.ts` 读取 `VITE_WEB_AUTH_TOKEN`。
- REST 请求自动加：

```http
Authorization: Bearer <token>
```

- WebSocket URL 自动加：

```text
/ws?token=<token>
```

- 如果 `VITE_WEB_AUTH_TOKEN` 为空，仍允许本地 server 模式工作；但 Cloudflare mode 文档必须提示设置。

Wrangler：

- `packages/cloudflare/worker-configuration.d.ts` 增加 `WEB_AUTH_TOKEN`。
- 不要把真实 token 写入 `wrangler.jsonc`。

### Task 5：Cloudflare Worker 集成测试

安装：

```bash
pnpm --filter @crewden/cloudflare add -D @cloudflare/vitest-pool-workers vitest
```

新增：

```text
packages/cloudflare/vitest.config.ts
packages/cloudflare/test/hub.test.ts
```

测试覆盖：

- `GET /api/channels` 无 auth 返回 401。
- `GET /api/channels` 带正确 token 返回 `general`。
- `POST /api/agents` invalid body 返回 400。
- `POST /api/agents` valid body 创建 agent。
- daemon ready 后 `GET /api/machines` 能看到 online machine。
- 创建 agent 后，daemon websocket 能收到 `agent:start`。
- browser websocket 能收到 `message:new`。

注意：

- 测试环境使用固定 secret：

```text
WEB_AUTH_TOKEN=test-web-token
DAEMON_API_KEY=test-daemon-key
```

- 不依赖真实 Cloudflare 远端资源。
