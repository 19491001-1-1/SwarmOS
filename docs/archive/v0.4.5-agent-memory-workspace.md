# v0.4.5 — Agent Memory, Notes & Workspace Foundation

> **给 Claude Code 的执行指令：** 实现本文档描述的所有功能。按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。循环直到所有验收条件满足。完成后运行 `pnpm verify` 确认全绿。

---

## 背景

一套成熟 daemon 实现的长期可用性依赖三个本地持久层：

1. 每个 agent 独立 workspace
2. `MEMORY.md` 作为长期身份和上下文入口
3. `notes/` 保存用户偏好、channel 背景、工作日志等结构化长期知识

crewden 当前已有 workspace/browser 的规划，但缺少“agent 自己如何使用 workspace 与 memory”的基础约束。v0.4.5 先补 daemon 侧 foundation，v0.5 再做 UI 文件浏览器。

---

## 目标

1. 每个 agent 有稳定的本地 workspace 根目录
2. daemon 启动 agent 前自动创建 `MEMORY.md` 与 `notes/`
3. standing prompt 明确要求 agent 维护 memory/notes
4. workspace 路径隔离，后续 v0.5 文件浏览 API 可直接复用
5. 不引入复杂 UI，只保证本地持久上下文可用

---

## Workspace 结构

默认：

```text
~/.crewden/agents/<agentId>/
  MEMORY.md
  notes/
    user-preferences.md
    channels.md
    work-log.md
  .crewden/
    agent-token
    crewden
```

允许通过环境变量覆盖根目录：

```text
CREWDEN_AGENTS_DIR=/custom/path
```

---

## MEMORY.md 初始模板

```markdown
# <Agent Display Name>

## Role
<description or default role>

## Key Knowledge
- No notes yet.

## Active Context
- First startup.

## Collaboration
- Use `crewden` CLI for channel messages, DMs, and delegation when available.
- Keep durable notes in `notes/`.
```

---

## Standing Prompt 增强

追加内容：

- 当前 cwd 是持久 workspace
- 每次重要任务后更新 `MEMORY.md` 或 `notes/work-log.md`
- 用户偏好写入 `notes/user-preferences.md`
- channel 背景写入 `notes/channels.md`
- 不要把 token、密钥、个人隐私写入普通 notes
- 上下文压缩或重启后优先读取 `MEMORY.md`

## 实现任务

### Task 1：Workspace 管理模块

新增：

```text
packages/daemon/src/workspace/
```

实现：

- `getAgentWorkspaceRoot(agentId)`
- `ensureAgentWorkspace(agent)`
- `ensureMemoryFile(agent)`
- `ensureNotes(agent)`
- `safeResolveAgentPath(agentId, relPath)`

安全要求：

- 禁止 `..` 路径穿越
- 禁止软链接逃逸，至少在 read API 层通过 realpath 校验
- 默认不删除用户文件

### Task 2：AgentProcessManager 集成

- `startAgent()` 前调用 `ensureAgentWorkspace`
- driver spawn cwd 使用 agent workspace
- 与 v0.4.3 的 `.crewden` CLI 注入兼容

### Task 3：Prompt 集成

- `packages/daemon/src/drivers/*`
  - standing prompt 中加入 memory/workspace 指令
  - 确保不同 runtime 都收到相同核心规则

### Task 4：为 v0.5 暴露 daemon 方法

先只在 daemon 内实现，不要求 server API：

```ts
readWorkspace(agentId, relPath): Promise<WorkspaceEntry>
```

返回：

```ts
type WorkspaceEntry =
  | { type: 'dir'; path: string; children: WorkspaceFile[] }
  | { type: 'file'; path: string; content: string; truncated?: boolean; binary?: boolean };
```

限制：

- 文本文件最大读取 1MB
- 二进制文件不返回 content
