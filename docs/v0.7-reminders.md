## 目标

Agent 可以通过 bridge 协议设置定时提醒，提醒触发时自动向指定 channel 发送消息。用户也可以手动为 agent 设置提醒。

---

## 参考截图描述

- REMINDERS tab：空状态显示铃铛图标 + "No pending reminders." + 说明文字
- 有提醒时：列表显示每条提醒的触发时间、内容、状态（Pending/Done/Cancelled）

---

## 数据模型

```ts
type Reminder = {
  id: string;
  agentId: string;
  channelId: string;
  message: string;          // 触发时发送的内容
  triggerAt: string;        // ISO 时间
  status: 'pending' | 'triggered' | 'cancelled';
  createdAt: string;
};
```

---

## Bridge 协议扩展

在 system prompt 中新增：
```
To schedule a reminder, output:
[[MINI_SLOCK_SET_REMINDER]] {"message":"...","triggerAt":"<ISO8601>","channelId":"<channelId>"}
To cancel a reminder, output:
[[MINI_SLOCK_CANCEL_REMINDER]] {"reminderId":"..."}
```

## 实现任务

### Task 1：协议 & 验证

- `packages/shared/src/protocol.ts`：添加 `Reminder` 类型
- `packages/shared/src/validation.ts`：对应 Zod schema
- `packages/daemon/src/bridge/simpleToolBridge.ts`：添加 `parseReminderLine()` 和 `parseCancelReminderLine()`
- Browser 推送新增：`{ type: 'reminder:update'; reminder: Reminder }`

### Task 2：Server 存储 & API

- `packages/server/src/db.ts`：添加 `reminders` 表
- 新增端点：
  ```
  GET    /api/agents/:id/reminders          # 列出该 agent 的提醒
  POST   /api/agents/:id/reminders          # 手动创建提醒
  PATCH  /api/reminders/:id                 # 取消提醒
  ```
- **定时触发器**：server 启动时启动一个 `setInterval`（每 10 秒），扫描 `triggerAt <= now` 且 `status = pending` 的提醒，触发后：
  1. 将提醒内容作为 agent 消息发送到对应 channel
  2. 更新 reminder status 为 `triggered`
  3. 推送 `reminder:update` 给 browser
- 测试：`packages/server/test/remindersApi.test.ts`

### Task 3：Daemon 解析

- `packages/daemon/src/agentProcessManager.ts`：
  - 检测 `[[MINI_SLOCK_SET_REMINDER]]` → 发送 `agent:set_reminder` 给 server
  - 检测 `[[MINI_SLOCK_CANCEL_REMINDER]]` → 发送 `agent:cancel_reminder` 给 server
- `packages/daemon/src/daemonClient.ts`：处理新消息类型

### Task 4：Web UI

- `AgentDetailPanel.tsx` REMINDERS tab：
  - 空状态：铃铛图标 + 文字（像素风虚线边框）
  - 有提醒时：列表，每行显示触发时间 + 内容 + 状态 badge
  - 手动添加提醒按钮（输入消息 + 时间选择器）
  - 实时通过 WebSocket 更新（`reminder:update` 事件）
