# v1.5 — Knowledge & Memory Layer

> **给 Claude Code / Codex 的执行指令：** 实现本文档描述的 v1.5。按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。完成后运行 `pnpm verify`；如果触及 Cloudflare Worker，也必须运行 `pnpm --filter @crewden/cloudflare exec wrangler deploy --dry-run`。如果触及 Web 构建，也运行 `VITE_API_BASE=https://crewden-hub.xingke0.workers.dev pnpm --filter @crewden/web build`。

## 背景

当前 Agent 有个人 `MEMORY.md` 和 workspace notes，但知识仍是分散的。v1.5 要把项目知识、决策、用户偏好、runbook、经验和产物沉淀成可检索、可引用、可复用的知识层。

v1.5 先做内置知识层和 adapter interface，不绑定单一外部知识库。默认 adapter 使用本地/仓库 Markdown，后续可以接 Notion、Drive、GitHub Wiki、Obsidian 或向量库。

## 目标

- Agent 执行任务前能检索相关知识摘要。
- Goal/Task 完成后能生成 project archive。
- 用户能搜索 decision、runbook、learning、artifact。
- 知识条目有来源、owner、reviewer、stale/conflict 标记。
- 定义 Knowledge Adapter Interface，后续外部知识库可接入。

## 非目标

- 不在 v1.5 绑定 Notion/Drive 等单一产品。
- 不把向量库当作唯一事实源。
- 不做复杂权限系统，先沿用当前 workspace/server 权限边界。
- 不自动把所有聊天都沉淀为知识，避免噪声污染。

## 数据模型

```ts
type KnowledgeKind =
  | 'decision'
  | 'project_archive'
  | 'user_preference'
  | 'runbook'
  | 'learning'
  | 'artifact';

type KnowledgeStatus = 'active' | 'stale' | 'conflict' | 'archived';

type KnowledgeEntry = {
  id: string;
  kind: KnowledgeKind;
  title: string;
  summary: string;
  body: string;
  tags: string[];
  sourceRefs: string[];
  ownerAgentId?: string;
  reviewerAgentId?: string;
  status: KnowledgeStatus;
  createdAt: string;
  updatedAt: string;
};

type KnowledgeSearchResult = {
  entry: KnowledgeEntry;
  score?: number;
  reason?: string;
};
```

Adapter interface：

```ts
type KnowledgeAdapter = {
  id: string;
  name: string;
  search(query: string, options?: { tags?: string[]; kind?: KnowledgeKind }): Promise<KnowledgeSearchResult[]>;
  read(id: string): Promise<KnowledgeEntry | undefined>;
  write(entry: KnowledgeEntry): Promise<KnowledgeEntry>;
  update(id: string, patch: Partial<KnowledgeEntry>): Promise<KnowledgeEntry>;
};
```

## 实现任务

### Task 1：Shared protocol & validation

- 增加 `KnowledgeEntry`、`KnowledgeKind`、`KnowledgeStatus`。
- 增加 create/update/search schema。
- Browser event：

```ts
{ type: 'knowledge:update'; entry: KnowledgeEntry }
```

测试：`packages/shared/test/protocol.test.ts`

### Task 2：Storage and API

Server 和 Cloudflare 增加：

```http
GET    /api/knowledge?query=&kind=&tag=
POST   /api/knowledge
GET    /api/knowledge/:id
PATCH  /api/knowledge/:id
POST   /api/goals/:goalId/archive
```

要求：

- 搜索第一版可以用关键词匹配 title/summary/body/tags，不要求向量检索。
- `archive` 从 Goal、Task、Review、Evidence 生成 project archive 草案。
- 条目必须有 sourceRefs，除非用户明确手动创建无来源条目。

### Task 3：Markdown/Git adapter

新增内部 adapter：

- 本地 server/daemon 可写 Markdown 文件。
- Cloudflare 先使用 Durable Object SQLite 作为内置 adapter。
- Markdown 格式采用 frontmatter + body：

```md
---
id: ...
kind: decision
tags: [architecture, v1]
status: active
sourceRefs: [...]
---

正文
```

不要求一开始把 Cloudflare 写回 Git repo。

### Task 4：Agent internal API & CLI

Internal API：

```http
GET  /internal/agent/:agentId/knowledge?query=&kind=&tag=
GET  /internal/agent/:agentId/knowledge/:id
POST /internal/agent/:agentId/knowledge
POST /internal/agent/:agentId/goals/:goalId/archive
```

CLI：

```bash
crewden knowledge search "query"
crewden knowledge read <id>
crewden knowledge write --kind decision --title "..." --summary "..." --body "..." --tag v1
crewden goal archive <goalId>
```

要求：

- Agent 执行任务前优先 `knowledge search`。
- `goal archive` 输出 archive entry id。

### Task 5：Bridge prompt

更新 Agent 指令：

- 执行有上下文依赖的任务前检索知识。
- 新决策、用户偏好、失败教训、可复用 runbook 要写入 knowledge。
- 不要把短期闲聊写入 knowledge。
- 如果检索到冲突知识，标记 conflict 或升级给用户。

### Task 6：Web UI

新增轻量 Knowledge 入口，可以放在右侧 rail 或 Tasks/Agent detail 中：

- search box
- kind/tag filters
- entry detail
- create/edit entry
- stale/conflict 标记
- Goal archive 入口

Agent detail 展示该 Agent 最近创建/使用的 knowledge。

## 手动验证

1. 创建一条 decision：“v1 测试环境使用 crewden-hub-v1”。
2. 用 `crewden knowledge search "测试环境"` 搜到该条目。
3. 创建一个 Goal 并完成若干任务。
4. 执行 `crewden goal archive <goalId>`。
5. 在 Web Knowledge 搜索 project archive。
6. 修改条目为 stale，确认 Web 和 CLI 都能看到状态。
