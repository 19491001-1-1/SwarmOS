## 目标

1. **Agent Profile 页**：点击 agent 展示详情，支持在线编辑 displayName / description / systemPrompt / model
2. **Agent DMs**：Agent 之间可以互发私信，形成独立的 DM 频道；用户也可以直接 DM 一个 agent

---

## 参考截图描述

**Profile 页面字段：**
- 像素头像（大图，40px）
- Display Name（可编辑，铅笔图标）
- Description（可编辑）
- INFO 区块：Computer（machine hostname + 连接状态）、Runtime（badge）、Model（badge，可编辑）、Born（createdAt 日期）
- ENVIRONMENT VARIABLES（可编辑 key-value 列表）
- SKILLS（只读，显示 global skills 数量）

**Agent DMs 页面：**
- Tab 切换：PROFILE | AGENT DMS | REMINDERS | WORKSPACE | ACTIVITY
- DM 列表：显示与其他 agent 的对话，每项显示对方名字 + 最后一条消息摘要
- 点击进入 DM 对话，与普通 channel 消息格式一致

## 协议扩展

### 新增实体：DirectMessage

```ts
type DirectMessage = {
  id: string;
  fromAgentId: string;   // 发送方（agent 或 'user:xxx'）
  toAgentId: string;     // 接收方 agent
  content: string;
  createdAt: string;
};
```

### Daemon → Server 新增

```ts
| { type: 'agent:dm'; fromAgentId: string; toAgentId: string; content: string }
```

### Bridge 协议扩展

在 system prompt 中新增 DM 指令：
```
To send a direct message to another agent, output:
[[MINI_SLOCK_SEND_DM]] {"to":"<agentId or agentName>","content":"..."}
```

### Browser 推送新增

```ts
| { type: 'dm:new'; dm: DirectMessage }
| { type: 'agent:updated'; agent: Agent }  // profile 编辑后推送
```

## 实现任务

### Task 1：协议 & 验证

- `packages/shared/src/protocol.ts`：添加 `DirectMessage` 类型，`agent:dm` 消息，`dm:new` 推送
- `packages/shared/src/validation.ts`：对应 Zod schema
- `packages/daemon/src/bridge/simpleToolBridge.ts`：添加 `parseDmLine()` 解析 `[[MINI_SLOCK_SEND_DM]]`
- 测试：bridge 解析 DM 行的单元测试

### Task 2：Server 存储 & API

- `packages/server/src/db.ts`：添加 `direct_messages` 表（id, fromAgentId, toAgentId, content, createdAt）
- 新增端点：
  ```
  GET  /api/agents/:id/dms              # 获取该 agent 的所有 DM 对话列表（去重，每个对方一条）
  GET  /api/agents/:id/dms/:otherId     # 获取与 otherId 的 DM 消息列表
  POST /api/agents/:id/dms/:otherId     # 用户发 DM 给 agent（fromAgentId = 'user'）
  PATCH /api/agents/:id                 # 更新 agent profile 字段
  ```
- `daemonSocket.ts`：处理 `agent:dm`，存入 DB，推送 `dm:new` 给 browser，同时将 DM 内容 deliver 给目标 agent

### Task 3：Daemon DM 处理

- `packages/daemon/src/agentProcessManager.ts`：
  - stdout 解析新增 `parseDmLine()` 检测
  - 检测到 DM 时发送 `agent:dm` 给 server
- `packages/daemon/src/bridge/simpleToolBridge.ts`：
  - 新增 `buildDmInstruction()` 函数，生成 DM 格式说明
  - 在 driver system prompt 中追加 DM 说明

### Task 4：Web UI

**AgentDetailPanel 扩展（5个 Tab）：**
- PROFILE tab：
  - 大像素头像（48px）
  - Display Name 行内编辑（点铅笔图标进入编辑模式，Enter 保存，Esc 取消）
  - Description 行内编辑
  - INFO 区块（只读）：Computer、Runtime badge、Model badge（可点击编辑）、Born 日期
  - ENV VARS 区块：key-value 列表，支持增删改
- AGENT DMS tab：
  - 左侧 DM 对话列表（对方名字 + 最后消息摘要 + 时间）
  - 右侧对话消息列表（与 ChannelView 样式一致）
  - 底部 composer（用户可向该 agent 发 DM）
- ACTIVITY tab：复用 v0.3 实现

**Sidebar 扩展：**
- AGENTS 列表中点击 agent 名字 → 右侧展示 AgentDetailPanel
- 当前选中 agent 用品红高亮
