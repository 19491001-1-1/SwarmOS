## 目标

实现 Kanban 看板。用户可以将 channel 消息转为任务，任务有 TODO / IN PROGRESS / IN REVIEW / DONE 四个状态列，支持拖拽移动（可选）和手动状态切换。

---

## 参考截图描述

- 顶部：`Tasks` 标题 + `N channel tasks` 计数 + `Board` / `List` 切换按钮
- 顶部筛选：`# CHANNEL ▼` 下拉（按 channel 筛选）
- 四列看板：
  - **TODO**（橙色标签）
  - **IN PROGRESS**（青色标签）
  - **IN REVIEW**（紫色标签）
  - **DONE**（绿色标签）
- 每张卡片：
  - 来源 channel + 任务编号（`#all #3`）
  - 任务内容（消息正文，超长截断）
  - `creator @xxx` + `assignee @xxx`
- 空列：虚线边框 + "No xxx tasks."
- Sidebar：Tasks 作为顶级导航项

---

## 数据模型

```ts
type Task = {
  id: string;
  channelId: string;
  messageId?: string;       // 来源消息（可选，手动创建的任务无来源）
  title: string;            // 任务内容（来自消息正文，最多 200 字符）
  status: 'todo' | 'in_progress' | 'in_review' | 'done';
  creatorName: string;      // 创建人（用户名或 agent 名）
  assigneeId?: string;      // 分配给哪个 agent
  createdAt: string;
  updatedAt: string;
};
```

## 实现任务

### Task 1：协议 & 验证

- `packages/shared/src/protocol.ts`：添加 `Task` 类型与 agent-only `TaskContext`
- `packages/shared/src/validation.ts`：对应 Zod schema
- Browser 推送新增：`{ type: 'task:update'; task: Task }`

`Task` 只承载 Web UI 需要直接展示的字段；`TaskContext` 承载 agent 执行和交接所需的信息，默认不需要在任务卡片上展示：

```ts
type TaskContext = {
  goal?: string;
  background?: string;
  acceptanceCriteria?: string[];
  constraints?: string[];
  sourceMessageIds?: string[];
  artifacts?: string[];
  requesterAgentId?: string;
  previousAgentId?: string;
  handoffNotes?: string[];
  privateNotes?: string[];
};
```

### Task 2：Server 存储 & API

- `packages/server/src/db.ts`：添加 `tasks` 表
- 新增端点：
  ```
  GET    /api/tasks?channelId=&status=        # 列出任务（支持筛选）
  POST   /api/tasks                           # 创建任务
  PATCH  /api/tasks/:id                       # 更新状态/assignee/context
  DELETE /api/tasks/:id                       # 删除任务
  POST   /api/messages/:id/to-task            # 消息转任务
  ```
- 测试：`packages/server/test/tasksApi.test.ts`

Agent internal API additionally supports:

```bash
GET  /internal/agent/:agentId/agents/resolve?query=
GET  /internal/agent/:agentId/tasks
GET  /internal/agent/:agentId/tasks/:taskId
POST /internal/agent/:agentId/tasks/:taskId/update
POST /internal/agent/:agentId/tasks/:taskId/handoff
```

`agents/resolve` lets agents map user-facing names like `产品经理`, display names, or role-description hints to a concrete `agentId` before DM, delegation, or task handoff.

`handoff` updates `assigneeId`, records `previousAgentId`, and appends a structured note to `context.handoffNotes`.

### Task 3：Bridge 协议扩展（Agent 创建任务）

```
To create a task, output:
[[MINI_SLOCK_CREATE_TASK]] {"title":"...","assignee":"<agentName>","channel":"<channelId>"}

[[MINI_SLOCK_UPDATE_TASK]] {"taskId":"...","status":"todo|in_progress|in_review|done"}
```

- `packages/daemon/src/bridge/simpleToolBridge.ts`：添加解析函数
