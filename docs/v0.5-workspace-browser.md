## 实现任务

### Task 1：Server API

新增端点（daemon 代理文件系统访问）：

```
GET /api/agents/:id/workspace?path=<relPath>
```

- 返回目录列表（type: 'dir', children）或文件内容（type: 'file', content）
- `relPath` 为相对于 agent workspace 根目录的路径，默认 `""`（根）
- 安全限制：禁止路径穿越（`..`），只允许在 agent workspace 目录内访问
- 文件大小限制：超过 100KB 返回截断提示

实际文件读取由 daemon 执行：
- Server 收到请求 → 通过 daemon WebSocket 发送 `workspace:read` 指令
- Daemon 读取文件系统 → 回复 `workspace:result`

### Task 2：协议扩展

```ts
// Server → Daemon
| { type: 'workspace:read'; agentId: string; requestId: string; relPath: string }

// Daemon → Server
| { type: 'workspace:result'; requestId: string; result: WorkspaceEntry | WorkspaceError }

type WorkspaceEntry =
  | { type: 'dir'; path: string; children: WorkspaceFile[] }
  | { type: 'file'; path: string; content: string; truncated?: boolean };

type WorkspaceFile = {
  name: string;
  type: 'file' | 'dir';
  size?: number;
  modifiedAt?: string;
};
```

### Task 3：Daemon 实现

- `packages/daemon/src/daemonClient.ts`：处理 `workspace:read`，使用 `fs/promises` 读取文件系统
- `packages/daemon/src/agentProcessManager.ts`：新增 `readWorkspace(agentId, relPath)` 方法

### Task 4：Web UI

- `packages/web/src/api.ts`：添加 `getAgentWorkspace(agentId, relPath?)` 函数
- 新建 `packages/web/src/components/WorkspaceBrowser.tsx`：
  - 左侧文件树（可展开文件夹，点击文件加载内容）
  - 右侧预览区（monospace，pre 标签，内容截断时显示提示）
  - 顶部路径面包屑 + 刷新按钮
  - 像素风样式（border 2px solid #000，无圆角）
- `AgentDetailPanel.tsx`：在 WORKSPACE tab 中渲染 WorkspaceBrowser
