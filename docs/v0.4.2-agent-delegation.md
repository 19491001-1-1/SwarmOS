## 背景

v0.4 已经实现了 Agent Profile 与 DM，但当前设计只支持 agent “发私信”，不支持 agent “主动委派并拉起另一个 agent 干活”。

这会导致一个协作断点：

- Agent A 可以输出 `[[CREWDEN_SEND_DM]]` 给 Agent B
- 如果 Agent B inactive，DM 只会存储，不会启动 B
- Agent A 没有协议表达“请启动 B 并交给它这个任务”

参考一套成熟 daemon 实现的设计，云端可以通过 `agent:deliver` 在目标 agent 未运行时唤醒它，并把 wake message 放进启动 prompt。v0.4.2 目标是补齐这个最关键的协作能力。

---

## 目标

1. **Agent 可以主动委派任务给另一个 agent**
2. **目标 agent inactive 时，hub/server 能按需启动它**
3. **委派请求可审计**：持久化 delegation/DM/activity，而不是只做瞬时投递
4. **本地 server 与 Cloudflare hub 行为一致**
5. **不把普通 DM 隐式升级为启动请求**，启动必须来自显式 delegate 协议

---

## 协议扩展

### Daemon → Hub

```ts
| {
    type: 'agent:delegate';
    fromAgentId: string;
    toAgentId: string;          // agentId or agentName
    content: string;
    startIfInactive?: boolean;  // default true
  }
```

### Browser 推送

```ts
| { type: 'agent:delegation'; delegation: AgentDelegation }
```

### 新增实体

```ts
type AgentDelegation = {
  id: string;
  fromAgentId: string;
  toAgentId: string;
  content: string;
  status: 'queued' | 'delivered' | 'started' | 'failed';
  error?: string;
  createdAt: string;
};
```

### Bridge 指令

在 runtime system prompt 中追加：

```text
To delegate work to another agent and wake it if needed, output exactly one line:
[[CREWDEN_DELEGATE_AGENT]] {"to":"agentId or agentName","content":"task details","startIfInactive":true}
```

---

## Hub 行为语义

收到 `agent:delegate` 后：

1. 按 id/name resolve `toAgentId`
2. 创建 `AgentDelegation` 记录，初始 `queued`
3. 同时创建一条 DirectMessage，便于 DM 历史中看到委派内容
4. 如果目标 agent 当前 `inactive/error/disconnected` 且 `startIfInactive !== false`：
   - 选择可运行该 runtime 的 machine
   - 发送 `agent:start`
   - `wakeMessage` 内容来自 delegation
   - delegation 状态更新为 `started`
5. 如果目标 agent 已 running/working/idle：
   - 发送 `agent:deliver`
   - delegation 状态更新为 `delivered`
6. 如果没有可用 machine：
   - delegation 状态更新为 `failed`
   - 记录 `error`
   - 不丢失 DirectMessage
