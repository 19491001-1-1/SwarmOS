# v1.0 — Organization Graph

> **给 Claude Code 的执行指令：** 实现本文档描述的 v1.0 窄切面。按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。完成后运行 `pnpm verify`；如果触及 Cloudflare Worker，再运行 wrangler dry-run。

## 目标

把 agent 从“可聊天对象”升级为“组织成员”。本版本先落地可验证的组织画像基础，让 role、department、capability 能进入 agent directory 和 resolve/delegation 流程。

## 数据模型

在 `Agent` 上新增 `organization`：

```ts
{
  department?: string;
  roles?: string[];
  capabilities?: string[];
  responsibilities?: string[];
  managerId?: string;
  backupAgentIds?: string[];
  availability?: 'available' | 'unavailable' | 'overloaded';
}
```

## 实现范围

- Shared protocol/validation 支持 `Agent.organization`
- Server SQLite/Cloudflare Durable Object 持久化 `organization`
- `POST /api/agents` 和 `PATCH /api/agents/:id` 可写入组织画像
- `xoxiang agent list` 通过 server info 返回组织画像
- `xoxiang agent resolve` 匹配 name/displayName 后，继续匹配 department、roles、capabilities、responsibilities
- 保持现有 delegation/handoff 复用 resolve 行为

## 验收条件

- 用户能给 agent 设置部门、角色、能力、职责、manager、backup、availability
- 输入角色或能力关键词时，resolve 能返回对应 agent 或候选列表
- `pnpm verify` 全绿
- Cloudflare Worker dry-run 通过

## 不做的事

- 不在 v1.0 引入独立 Department/Role 表
- 不实现完整组织架构图 UI
- 不实现负载调度策略，只持久化 availability 字段供后续版本使用
