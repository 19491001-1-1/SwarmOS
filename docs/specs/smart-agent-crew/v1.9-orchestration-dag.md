# v1.9 Orchestration DAG Roadmap

## 版本目标

把 v1.6 的 dependency graph 升级为可执行编排 DAG：支持多 agent 并行、汇合等待、条件检查点、workspace semantic memory、自我改进循环和多 daemon 协作。

## 非目标

- 不做复杂低代码工作流平台。
- 不做企业多租户权限矩阵。
- 不要求第一版 UI 完整可视化；API/协议先行。

## 为什么现在做

当 Crewden 有可靠状态机、plan/verify、memory/skill 后，瓶颈会变成“多个 agent 如何协作完成复杂目标”。v1.9 让 Crewden 从 task handoff 走向 agent crew orchestration。

## 核心用户故事

1. 作为用户，我提交“迁移 CI 并更新文档”，系统拆成并行节点：分析、实现、测试、文档、review。
2. 作为 coding agent，我只在依赖满足后执行，并能把结果传给下游 agent。
3. 作为系统，我能按机器/runtime 能力把任务分发给最合适 daemon。
4. 作为 owner，我能在高风险节点设置 human checkpoint。

## 数据模型/API/协议建议

- 新表/模型：`workflow_runs`、`workflow_nodes`、`workflow_edges`、`workflow_node_events`。
- Node fields：`status, taskId, agentId, machineId, runtime, condition, retryPolicy, checkpointPolicy`。
- Edge fields：`fromNodeId, toNodeId, type: success|failure|always|manual`。
- API：`POST /goals/:id/workflow`、`POST /workflow-runs/:id/start`、`POST /workflow-nodes/:id/approve`。
- Protocol：`workflow:update` websocket event。

## TDD 任务拆解

### Task 1：Workflow DAG schema 与循环检测

- Failing test：`rejects cyclic workflow graph` in `goalsApi.test.ts`。
- 命令：`pnpm --filter @crewden/server test -- goalsApi.test.ts -t "cyclic workflow"`
- 期望失败：无 workflow graph API。
- 最小实现：新增 workflow 数据结构和 topological validation；只存储，不执行。
- 通过标准：DAG 创建成功，cycle/self-loop 422。
- Commit：`feat(workflow): add dag schema validation`

### Task 2：DAG 执行调度器最小版

- Failing test：`starts only root nodes and waits for dependencies`。
- 命令：`pnpm --filter @crewden/server test -- goalsApi.test.ts -t "root nodes"`
- 期望失败：无调度器。
- 最小实现：`taskDelivery.ts` 或新 `workflowScheduler.ts` 根据 node 入度启动 root；node done 后检查下游。
- 通过标准：并行 root 同时投递；join 节点等待所有上游 success。
- Commit：`feat(workflow): schedule dag nodes by dependency`

### Task 3：Human-in-the-loop checkpoints

- Failing test：`pauses checkpoint node until approved`。
- 命令：`pnpm --filter @crewden/server test -- goalsApi.test.ts -t "checkpoint"`
- 期望失败：checkpoint 无效。
- 最小实现：node policy `requiresApproval`; 调度到节点时状态 `waiting_approval`; approve API 后投递。
- 通过标准：未 approve 不投递；approve 写 audit 并继续。
- Commit：`feat(workflow): add human checkpoints`

### Task 4：Capability-aware daemon routing

- Failing test：`routes node to daemon with required runtime capability` in `daemonSocket.test.ts`。
- 命令：`pnpm --filter @crewden/server test -- daemonSocket.test.ts -t "runtime capability"`
- 期望失败：routing 不看 capability。
- 最小实现：daemon registry 记录 runtime/capabilities；taskDelivery 选择匹配 machine。
- 通过标准：需要 codex 的 node 不发给只有 claude 的 daemon。
- Commit：`feat(daemon): route workflow nodes by capability`

### Task 5：Workspace semantic memory 接口

- Failing test：`stores and retrieves workflow memory references`。
- 命令：`pnpm --filter @crewden/server test -- knowledgeApi.test.ts -t "workflow memory"`
- 期望失败：无 workspace memory kind。
- 最小实现：先不引入向量 DB；定义接口 `SemanticMemoryProvider` 和 keyword fallback；knowledge scope 支持 `workspace:<id>`。
- 通过标准：后续可替换 sqlite-vec/Vectorize；当前测试用 fake provider。
- Commit：`feat(memory): add workspace semantic memory provider interface`

### Task 6：Agent self-improvement loop

- Failing test：`records improvement note from failed then successful retry`。
- 命令：`pnpm --filter @crewden/server test -- tasksApi.test.ts -t "improvement note"`
- 期望失败：没有从失败/成功 diff 生成学习记录。
- 最小实现：任务完成后读取 retry/audit/verification，生成 `kind: memory` 草稿，scope `agent:<id>`，内容含“下次遇到 X 先做 Y”。
- 通过标准：只生成草稿；敏感信息 redacted；重复任务去重。
- Commit：`feat(agent): draft improvement memories from task outcomes`

## 测试用例清单

- Happy：DAG root 并行、join 等待、checkpoint approve 后执行、capability routing 命中。
- Edge：空 DAG、单节点 DAG、多个 root、节点取消、上游 failure edge。
- Failure：循环图、无可用 daemon、checkpoint 拒绝、semantic provider 不可用。
- Concurrency/restart：scheduler restart 后继续；两个 daemon 同时可用时只投递一次；node done 事件重复上报幂等。

## 验收标准

- Workflow API 可用，DAG 验证严格。
- Scheduler 所有动作可 audit，可重启恢复。
- 不依赖 UI 也能用 API/CLI 创建和运行 workflow。
- 高风险 checkpoint 默认安全，不能绕过状态机。

## 风险与降级

- DAG 复杂度失控：第一版只支持 success/all dependencies，不做条件表达式语言。
- Routing 错误：无匹配 daemon 时 block + audit，不 fallback 到任意 daemon。
- Semantic memory 成本：provider 接口先行，默认 keyword fallback。

## 版本依赖

依赖 v1.6 dependency/status/audit，v1.7 plan/checkpoint 原语，v1.8 memory/runtime capability。
