# v0.2.2 — CI/CD for Cloudflare Hub and Pages

> **给 Claude Code 的执行指令：** 实现本文档描述的所有功能。按 TDD/验证驱动工作：先让 CI 在本地可验证，再配置 GitHub Actions，再用 `gh` 检查 workflow 运行结果。循环直到所有验收条件满足。完成后运行 `pnpm verify` 确认全绿，并确保 CI 通过。不要停在方案设计，必须完成代码、配置、文档和提交。

---

## 目标

为 Xoxiang 建立 GitHub Actions CI/CD：

- PR / push 时自动 typecheck 和 test。
- push 到 `main` 时自动部署 Cloudflare Worker hub。
- push 到 `main` 时自动构建并部署 Cloudflare Pages web。
- 所有部署使用 GitHub Secrets，不在仓库中保存密钥。

---

## 背景

当前部署依赖本地命令：

```bash
pnpm --filter @mini-slock/cloudflare run deploy
pnpm deploy:web:pages
```

这对个人开发可用，但不适合作为长期交付路径。v0.2.2 要把 Cloudflare hub 和 Pages 静态 web 的发布流程放入 CI/CD。

---

## 前置要求

需要 GitHub Secrets：

```text
CLOUDFLARE_API_TOKEN
CLOUDFLARE_ACCOUNT_ID
DAEMON_API_KEY
```

建议 token 权限：

- Account: Cloudflare Pages Edit
- Account: Workers Scripts Edit
- Account: Workers Tail Read（可选）
- Account: Durable Objects Edit / Workers write 覆盖 DO migration
- Zone: DNS Read（如果 workflow 只部署，不管理 DNS，可以不需要 DNS Edit）

如果当前没有 token，Claude Code 应提示用户创建，并停止在“需要 secret”的位置；不要把真实 token 写入仓库。

## 实现任务

### Task 1：新增 CI workflow

文件：

```text
.github/workflows/ci.yml
```

触发：

```yaml
on:
  pull_request:
  push:
    branches: [main]
```

步骤：

- checkout
- setup pnpm
- setup Node.js 24
- pnpm install
- `pnpm verify`
- `pnpm --filter @mini-slock/cloudflare exec wrangler deploy --dry-run`

要求：

- 使用 pnpm store cache。
- 不需要 Cloudflare secret 即可跑 CI dry-run。
- CI 失败时能清楚看到哪个 package 失败。

### Task 2：新增 Cloudflare Worker deploy workflow

文件：

```text
.github/workflows/deploy-cloudflare-hub.yml
```

触发：

```yaml
on:
  push:
    branches: [main]
  workflow_dispatch:
```

步骤：

- checkout
- setup pnpm
- setup Node.js 24
- pnpm install
- `pnpm verify`
- 上传/更新 `DAEMON_API_KEY` secret 到 Cloudflare Worker：

```bash
printf '%s' "$DAEMON_API_KEY" | pnpm --filter @mini-slock/cloudflare exec wrangler secret put DAEMON_API_KEY
```

- 部署 Worker：

```bash
pnpm --filter @mini-slock/cloudflare run deploy
```

环境变量：

```yaml
CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
DAEMON_API_KEY: ${{ secrets.DAEMON_API_KEY }}
```

要求：

- 不在 logs 中打印 daemon key。
- 如果 `DAEMON_API_KEY` secret 缺失，workflow 应失败并给出明确错误。

### Task 3：新增 Cloudflare Pages deploy workflow

文件：

```text
.github/workflows/deploy-cloudflare-pages.yml
```

触发：

```yaml
on:
  push:
    branches: [main]
  workflow_dispatch:
```

步骤：

- checkout
- setup pnpm
- setup Node.js 24
- pnpm install
- build web：

```bash
VITE_API_BASE=https://xoxiang-hub.xingke0.workers.dev pnpm --filter @mini-slock/web build
```

- deploy Pages：

```bash
pnpm --filter @mini-slock/cloudflare exec wrangler pages deploy packages/web/dist \
  --project-name xoxiang-web \
  --branch main \
  --commit-dirty=true
```

环境变量：

```yaml
CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
```

要求：

- `VITE_API_BASE` 不要硬编码在源码里；可以在 workflow env 中集中定义。
- workflow 成功后输出 Pages URL。
