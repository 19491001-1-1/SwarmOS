## 目标

1. 用户可以创建和删除 channel（不止 `general`）
2. 全局消息搜索（跨 channel，按关键词）

## 实现任务

### Task 1：Channel 管理 API

新增端点：
```
POST   /api/channels              # 创建 channel（body: { name }）
DELETE /api/channels/:id          # 删除 channel（同时删除其所有消息和任务）
```

- `general` channel 不可删除（返回 400）
- channel name 唯一性校验（重复返回 409）
- 删除 channel 时推送 browser 事件：`{ type: 'channel:deleted'; channelId: string }`
- 测试：`packages/server/test/channelsApi.test.ts`

### Task 2：全局搜索 API

新增端点：
```
GET /api/search?q=<keyword>&limit=20
```

- 搜索 messages 表的 content 字段（LIKE %keyword%）
- 返回：`{ messages: (Message & { channelName: string })[] }`
- 结果按 createdAt desc 排序，最多返回 50 条
- 测试：`packages/server/test/searchApi.test.ts`

### Task 3：Web UI — Channel 管理

- Sidebar CHANNELS header 右侧添加 `+` 按钮（像素风，点击展开创建表单）
- 创建表单：输入框 + CONFIRM 按钮（Enter 确认，Esc 取消）
- Channel 列表项 hover 时显示 `✕` 删除按钮（general channel 不显示）
- 删除确认：简单 `window.confirm` 即可
- 实时通过 WebSocket 更新 channel 列表

### Task 4：Web UI — 搜索

- Sidebar 顶部添加 `SEARCH` 导航项（像素风，右侧显示 `⌘K`）
- 点击或按 `⌘K` / `Ctrl+K` 打开搜索面板（覆盖主区域）
- 搜索面板：
  - 顶部搜索输入框（autofocus，像素风大边框）
  - 输入 debounce 300ms 后触发搜索
  - 结果列表：每项显示 channel 名 + 消息内容（关键词高亮）+ 时间
  - 点击结果跳转到对应 channel
  - 按 Esc 关闭
- 空状态：`[ NO RESULTS ]`
