---

## 目标

用 SQLite 替换现有的内存 Map store，使 server 重启后数据不丢失。

---

## 背景

当前 `packages/server/src/db.ts` 使用纯内存 Map 实现。Node.js 24 上 `better-sqlite3` 编译失败，改用 `drizzle-orm` + `@libsql/client`（基于 libSQL/turso，无需 native binding，纯 JS）。

---

## 技术选型

- ORM：`drizzle-orm`
- 驱动：`@libsql/client`（file:// 本地 SQLite，无需 native binding）
- 迁移：`drizzle-kit`
- DB 文件路径：`~/.xoxiang/data.db`（可通过环境变量 `XOXIANG_DB_PATH` 覆盖）

---

## 实现任务

### Task 1：安装依赖

```bash
pnpm --filter @mini-slock/server add drizzle-orm @libsql/client
pnpm --filter @mini-slock/server add -D drizzle-kit
```

### Task 2：定义 Schema

文件：`packages/server/src/schema.ts`

```ts
// 表结构：
// channels: id, name, createdAt
// messages: id, channelId, senderName, content, agentId(nullable), createdAt
// agents: id, name, displayName, runtime, model, systemPrompt, machineId, status, createdAt
// machines: id, hostname, os, daemonVersion, runtimes(JSON), runtimeVersions(JSON), status, connectedAt
```

### Task 3：实现 db.ts

文件：`packages/server/src/db.ts`

- 导出 `getDb()` 函数，返回 drizzle 实例（单例）
- 启动时自动建表（`CREATE TABLE IF NOT EXISTS`）
- 初始化时插入默认 `general` channel（若不存在）
- 保持与现有 store 接口完全一致（`listChannels`, `getChannel`, `createChannel`, `listMessages`, `addMessage`, `listAgents`, `getAgent`, `createAgent`, `updateAgent`, `upsertMachine`, `listMachines`, `getMachine`）

### Task 4：更新 server 启动

文件：`packages/server/src/index.ts`

- 启动时调用 `await initDb()` 初始化数据库

### Task 5：更新测试配置

文件：`packages/server/vitest.config.ts`

- 测试使用内存 SQLite（`:memory:`），通过环境变量 `XOXIANG_DB_PATH=:memory:` 注入
- 保持 `isolate: false`（共享单例）
