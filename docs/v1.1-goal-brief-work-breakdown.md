# v1.1 — Goal Brief & Work Breakdown

> **给 Claude Code / Codex 的执行指令：** 实现本文档描述的 v1.1。执行时必须从 `main` 新建分支，按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。完成后运行 `pnpm verify`；如果触及 Cloudflare Worker，也必须运行 `pnpm --filter @crewden/cloudflare exec wrangler deploy --dry-run`。如果触及 Web 构建，也运行 `VITE_API_BASE=https://crewden-hub.xingke0.workers.dev pnpm --filter @crewden/web build`。

## 背景

v0.x 已经有聊天、任务板、Agent profile、delegation、workspace memory 和 agent-facing CLI。当前缺口是：用户一句“帮我做 X”无法稳定变成可执行的目标 brief、任务拆解、依赖、验收标准和 agent 可读上下文。

v1.1 先不做完整项目管理系统，也不做新的 Boss Dashboard。目标是把“聊天里的目标型请求”结构化成可以被 Agent 执行和复盘的 Goal Brief，并把工作拆到现有 Task Board。

## 目标

- 用户或 Agent 能从一条聊天消息创建 Goal Brief 草案。
- Goal Brief 能表达 objective、background、success criteria、constraints、assumptions、risks。
- Goal Brief 确认后能拆解成任务，任务带 owner、dependencies、acceptance criteria、artifacts。
- 现有 Task Board 的 `TaskContext` 能承载 goal/task 所需上下文。
- Agent 读取任务时能拿到足够上下文，不需要只靠聊天历史猜。

## 非目标

- 不新增复杂 Project 表。
- 不做甘特图、里程碑 UI、资源排期。
- 不做自动执行所有目标的 autonomous loop，v1.3 处理。
- 不做 review/acceptance workflow，v1.4 处理。
- 不要求 LLM 自动生成完美计划；v1.1 重点是数据结构和工作流落地。

## 数据模型

新增 `GoalBrief`：

```ts
type GoalBriefStatus = 'draft' | 'confirmed' | 'cancelled' | 'completed';

type GoalBrief = {
  id: string;
  channelId: string;
  sourceMessageId?: string;
  requesterName: string;
  objective: string;
  background: string[];
  successCriteria: string[];
  constraints: string[];
  assumptions: string[];
  risks: string[];
  status: GoalBriefStatus;
  createdAt: string;
  updatedAt: string;
};
```

扩展 `TaskContext`：

```ts
type TaskContext = {
  goalId?: string;
  goalObjective?: string;
  background?: string;
  acceptanceCriteria?: string[];
  constraints?: string[];
  assumptions?: string[];
  risks?: string[];
  dependencies?: string[];
  artifacts?: string[];
  sourceMessageIds?: string[];
  requesterAgentId?: string;
  previousAgentId?: string;
  handoffNotes?: string[];
  privateNotes?: string[];
};
```

说明：

- `GoalBrief` 是用户目标的结构化事实源。
- `TaskContext.goalId` 连接任务和目标。
- 任务卡片仍保持轻量展示，详细上下文默认给 Agent 和详情页使用。

## 实现任务

### Task 1：Shared protocol & validation

- 在 `packages/shared/src/protocol.ts` 增加 `GoalBrief`、`GoalBriefStatus`、`GoalTaskDraft` 类型。
- 在 `packages/shared/src/validation.ts` 增加 create/update schema。
- Browser WebSocket 增加：

```ts
{ type: 'goal:update'; goal: GoalBrief }
```

- 测试：`packages/shared/test/protocol.test.ts`

### Task 2：Server SQLite API

新增存储和 API：

```http
GET    /api/goals?channelId=&status=
POST   /api/goals
GET    /api/goals/:id
PATCH  /api/goals/:id
POST   /api/goals/:id/tasks
POST   /api/messages/:id/to-goal
```

`POST /api/goals/:id/tasks` 输入任务草案数组，创建现有 `Task`，并把 `goalId`、acceptance criteria、dependencies 写入 `context`。

测试：`packages/server/test/goalsApi.test.ts`

### Task 3：Cloudflare Durable Object parity

- 在 `packages/cloudflare/src/index.ts` 实现同等 goal API。
- 使用 Durable Object SQLite 持久化。
- 所有 browser auth 规则和现有 API 一致。
- 测试：`packages/cloudflare/test/hub.test.ts`

### Task 4：Agent internal API & CLI

Internal API：

```http
GET  /internal/agent/:agentId/goals
GET  /internal/agent/:agentId/goals/:goalId
POST /internal/agent/:agentId/goals
POST /internal/agent/:agentId/goals/:goalId/tasks
```

CLI：

```bash
crewden goal list --channel general --status draft
crewden goal read <goalId>
crewden goal create --channel general --objective "..." --success "..." --constraint "..."
crewden goal create-tasks <goalId> --tasks-json '[...]'
```

要求：

- CLI 输出 JSON，方便 Agent 解析。
- `goal read` 必须包含关联任务摘要。
- `goal create-tasks` 写入 `TaskContext.goalId`。

测试：`packages/daemon/test/agentCli.test.ts`

### Task 5：Bridge prompt update

更新 `packages/daemon/src/bridge/simpleToolBridge.ts`：

- 告诉 Agent 遇到目标型请求时先形成 brief。
- 告诉 Agent 不要直接创建大量任务，除非 brief 足够明确或用户确认。
- 提供 CLI 示例：

```bash
crewden goal create --channel general --objective "..." --success "..."
crewden goal create-tasks <goalId> --tasks-json '[...]'
```

测试：`packages/daemon/test/simpleToolBridge.test.ts`

### Task 6：Web UI

在现有聊天和任务板上做最小可用入口：

- Message hover action 增加 `→ Goal`。
- Goal draft panel 展示 objective、success criteria、constraints、risks。
- 用户可以编辑并 confirm/cancel。
- Goal confirm 后可以创建任务草案。
- Task detail 展示所属 goal 和 acceptance criteria。

不要求新增完整 Goal 页面，但可以在 Tasks 视图里增加 Goal filter。

测试：

- `packages/web/test/App.test.tsx`
- 新增 `packages/web/test/GoalBrief.test.tsx` 或对应组件测试。

## 验收条件

- [ ] 用户能把一条消息转成 Goal Brief draft。
- [ ] 用户能编辑并确认 Goal Brief。
- [ ] 确认后的 Goal 能创建多个任务。
- [ ] 创建的任务带 `context.goalId` 和 acceptance criteria。
- [ ] Agent CLI 能 list/read/create goal。
- [ ] Agent 读取任务时能看到 goal objective、背景、验收标准和约束。
- [ ] Server 和 Cloudflare 行为一致。
- [ ] `pnpm verify` 全绿。
- [ ] Cloudflare dry-run 通过。

## 手动验证

1. 在 `#general` 发送“帮我做一个外卖 App 的 MVP 方案”。
2. 点击消息 hover 的 `→ Goal`。
3. 编辑 brief，确认 success criteria。
4. 创建 3 个任务：产品方案、技术方案、验收测试。
5. 打开任务详情，确认能看到 goal context。
6. 用 agent CLI 执行 `crewden goal list` 和 `crewden goal read <goalId>`。
