## 背景

当前 xoxiang 的 agent 输出依赖 stdout marker：

- `[[MINI_SLOCK_SEND_MESSAGE]]`
- `[[MINI_SLOCK_SEND_DM]]`
- v0.4.2 会增加 `[[MINI_SLOCK_DELEGATE_AGENT]]`

这种方式可用但不完备：

- agent 不能主动查询消息历史
- agent 不能检查未读消息
- agent 不能列出其他 agent/channel/server 信息
- agent 无法通过工具式接口创建任务、领取任务、设置提醒
- stdout marker 难以承载复杂对象、附件、分页、错误处理

参考 `@slock-ai/daemon@0.40.2`，更先进的设计是 daemon 在 agent workspace 注入一个 `slock` CLI wrapper，并通过 agent token 调用 server 的 internal agent API。

---

## 目标

1. 为每个 agent 注入本地 `slock` CLI wrapper
2. Server/Cloudflare 提供 `/internal/agent/:agentId/...` API
3. Agent 可以通过 CLI 查询消息、发送消息、读取 DM、委派 agent
4. 使用 per-agent token，不把 daemon API key 暴露给 agent
5. 保留 stdout marker 作为兼容 fallback

---

## 设计原则

- Browser API 与 Agent Internal API 分离
- daemon API key 只用于 daemon 连接 hub
- agent token 只代表单个 agent
- internal API 必须校验：
  - `Authorization: Bearer <agentToken>`
  - `X-Agent-Id: <agentId>`
- agent token 存储在工作目录文件：
  - `<agent-workspace>/.xoxiang/agent-token`
- 不把 token 直接放进环境变量

---

## CLI 命令范围

本版本先实现协作最小集：

```bash
xoxiang auth whoami
xoxiang server info
xoxiang message send --channel general --content "..."
xoxiang message check
xoxiang message read --channel general --limit 20
xoxiang dm send --to agentName --content "..."
xoxiang agent delegate --to agentName --content "..." --start-if-inactive
```

后续版本再扩展：

- task create/claim/update
- reminder schedule/list/cancel
- attachment upload/view
- search/history pagination

## 实现任务

### Task 1：Shared 协议与 token 类型

- `packages/shared/src/protocol.ts`
  - 新增 `AgentTokenInfo`
  - 新增 internal API 响应类型：
    - `AgentWhoami`
    - `AgentServerInfo`
    - `AgentUnreadSummary`
- `packages/shared/src/validation.ts`
  - internal API request schema

### Task 2：Server Internal API

新增路由前缀：

```text
/internal/agent/:agentId/auth/whoami
/internal/agent/:agentId/server/info
/internal/agent/:agentId/messages/send
/internal/agent/:agentId/messages/check
/internal/agent/:agentId/messages/read
/internal/agent/:agentId/dms/send
/internal/agent/:agentId/delegate
```

实现要求：

- 所有 internal API 必须通过 agent token 认证
- token 与 agentId 必须匹配
- `server/info` 返回：
  - 当前 agent profile
  - channel 列表
  - 可见 agent 列表
  - hub version
- `messages/check` 返回未读摘要
- `messages/read` 返回最近历史
- `delegate` 复用 v0.4.2 的 delegation 逻辑

### Task 3：Cloudflare Internal API

- `packages/cloudflare/src/index.ts`
  - 同步实现 internal agent API
  - Durable Object 内持久化 agent token hash 或明文 token
  - 测试环境可使用确定性 token
- 注意 CORS：
  - internal API 不是 browser API，不需要允许任意 origin

### Task 4：Daemon Workspace 注入

- `packages/daemon/src/agentProcessManager.ts`
  - 启动 agent 前确保目录：
    - `.xoxiang/`
    - `.xoxiang/agent-token`
    - `.xoxiang/xoxiang`
  - wrapper `xoxiang` 放入 PATH 前缀
  - 环境变量：
    - `XOXIANG_AGENT_ID`
    - `XOXIANG_SERVER_URL`
    - `XOXIANG_AGENT_TOKEN_FILE`
  - 不注入 `XOXIANG_AGENT_TOKEN`

### Task 5：CLI 实现

新增 package 或 daemon 子模块：

```text
packages/daemon/src/cli/
```

最小实现：

- 解析 argv
- 读取 token 文件
- 调用 internal API
- stdout 输出稳定文本或 JSON
- 非 2xx 用非零 exit code

### Task 6：Standing Prompt 更新

runtime drivers 的 system prompt 从“只输出 marker”升级为：

1. 优先使用 `xoxiang` CLI
2. marker 仅作为 fallback
3. 说明 message header 与 DM/delegate 语义
4. 明确不要泄露 token 文件内容
