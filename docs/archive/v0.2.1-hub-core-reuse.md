## 目标

抽取 `packages/server` 和 `packages/cloudflare` 之间重复的 hub 业务规则，新增 `packages/hub-core`，让本地 Node server 和 Cloudflare Worker 共享纯业务逻辑，减少两套实现漂移。

---

## 背景

当前 Cloudflare 旁路已经可用，但 `packages/server/src/routes/*`、`packages/server/src/ws/daemonSocket.ts` 和 `packages/cloudflare/src/index.ts` 中重复实现了不少业务规则：

- agent runtime config 组装
- message → `AgentDelivery` 转换
- start agent 时选择可用 machine
- stale machine rebinding
- duplicate machine merge
- volatile status reset 规则
- channel/message/agent/machine row mapper

由于运行时不同，不能直接复用 Fastify route、Node `ws`、Drizzle/libSQL 或 Durable Object SQL。v0.2.1 的目标是复用“纯业务规则”，不是强行统一 runtime adapter。

---

## 设计原则

- `hub-core` 必须是纯 TypeScript，不能依赖 Fastify、Cloudflare Worker API、Drizzle、`@libsql/client`、`ws`。
- `hub-core` 可以依赖 `@crewden/shared`。
- `packages/server` 和 `packages/cloudflare` 继续保留自己的存储/连接适配层。
- 先抽稳定、低风险的逻辑；不要为了抽象而重写所有代码。
- 每抽一个函数，先写 core 单测，再改两边调用。

### Task 2：抽取 Agent 相关纯函数

在 `packages/hub-core/src/agent.ts` 实现：

```ts
import type { Agent, AgentRuntimeConfig, Machine } from '@crewden/shared';

export function toRuntimeConfig(agent: Agent): AgentRuntimeConfig;

export function resolveStartMachineId(input: {
  agent: Agent;
  machines: Machine[];
  connectedMachineIds: Set<string>;
}): string | undefined;

export function resetAgentStatusForRestart(status: Agent['status']): Agent['status'];
```

规则：

- `toRuntimeConfig` 保留 runtime、model、name、displayName、description、systemPrompt。
- `resolveStartMachineId` 优先使用 agent 当前绑定且 connected 的 machine。
- 当前绑定不可用时，选择第一个 connected 且 `machine.runtimes.includes(agent.runtime)` 的 machine。
- 没有兼容 machine 时返回 `undefined`。
- `starting/running/working/idle` 在 restart reset 后变为 `inactive`。
- `inactive/error` 保持不变。

### Task 3：抽取 Message 相关纯函数

在 `packages/hub-core/src/message.ts` 实现：

```ts
import type { AgentDelivery, Channel, Message } from '@crewden/shared';

export function toAgentDelivery(message: Message, channel: Channel): AgentDelivery;
```

规则：

- 字段完全匹配当前 server/cloudflare 对 daemon 的 delivery 格式。

### Task 4：抽取 Machine 相关纯函数

在 `packages/hub-core/src/machine.ts` 实现：

```ts
import type { Machine } from '@crewden/shared';

export function findExistingMachineId(input: {
  machines: Machine[];
  hostname: string;
  os: string;
}): string | undefined;

export function findDuplicateMachineIds(input: {
  machines: Machine[];
  targetMachineId: string;
  hostname: string;
  os: string;
}): string[];
```

规则：

- existing machine 用 `hostname + os` 匹配。
- duplicate 排除 `targetMachineId`。
