# v0.9.2 — Mobile Responsive Workspace

> 本文档是 v0.9.1 后的移动端自适应版本规划，可直接交给 Claude Code / Codex 执行。执行时从 `main` 新建 `feat/v0.9.2-mobile-responsive` 分支，按 TDD/截图验收推进，完成后运行 `pnpm verify`。

## 目标

让 Crewden 在手机和窄屏设备上可用，而不是桌面布局的缩小版。

v0.9.2 聚焦：

1. 左侧 sidebar 在移动端变成可打开/关闭的抽屉。
2. 右侧 Agent/Thread 面板在移动端变成 overlay，不挤压主聊天。
3. 主聊天、消息列表、Markdown 表格、Thread、Composer 在窄屏下不横向溢出。
4. 常用操作在触屏上可点击、可关闭、不会被浏览器底部区域遮挡。

## 依赖

- v0.9：聊天体验、Thread、Markdown、mention、presence。
- v0.9.1：右侧边栏按需出现、左侧 hover/active 动效。

如果 v0.9/v0.9.1 尚未完全实现，v0.9.2 仍应按最终目标设计，但实现时以当前代码可用状态为准，避免引入假 UI。

## 非目标

- 不重做移动端专属产品形态。
- 不新增离线模式。
- 不新增原生 App 能力。
- 不改 API 数据模型。
- 不把所有桌面高级操作都塞进手机首屏。

## 断点建议

使用清晰断点，不要用大量 magic number。

```ts
const BREAKPOINTS = {
  mobile: 767,
  tablet: 1023,
  desktop: 1024,
};
```

布局策略：

| 宽度 | 布局 |
|---|---|
| `>= 1024px` | 桌面布局：sidebar + main + optional right rail |
| `768px - 1023px` | tablet：sidebar 可收起，right rail overlay 或窄面板 |
| `< 768px` | mobile：单主视图，sidebar/right rail 都是 overlay drawer |

## 产品要求

### 1. 移动端 Top Bar

手机宽度下需要一个固定顶部栏：

- 左侧：菜单按钮，打开 sidebar drawer。
- 中间：当前频道/页面标题。
- 右侧：成员/Thread/更多入口，按当前上下文显示。

要求：

- 高度稳定，建议 `48px - 56px`。
- 不遮挡消息列表顶部。
- 标题长时截断。
- 点击菜单按钮后 sidebar drawer 打开。

### 2. Sidebar Drawer

移动端 sidebar 不常驻。

行为：

- 点击菜单按钮打开。
- 从左侧滑出或直接 overlay 出现。
- 带半透明遮罩。
- 点击遮罩、关闭按钮、ESC 关闭。
- 选择频道/Threads/Tasks 后自动关闭 drawer。

尺寸：

- 宽度建议 `min(86vw, 320px)`。
- 不得横向溢出。
- 内部列表可滚动。

验收：

- 375px 宽度下，sidebar 打开后完整可用。
- 长频道名和 DM 名不撑破 drawer。
- 关闭后主聊天恢复全宽。

### 3. Right Rail / Thread Mobile Overlay

移动端右侧面板不占主布局宽度。

行为：

- AgentPanel、ThreadPanel、详情面板都以 overlay sheet 展示。
- ThreadPanel 可以从右侧滑出，也可以全屏覆盖；建议手机用全屏覆盖更稳定。
- 顶部必须有返回/关闭按钮。
- 发送 Thread 回复后，键盘弹出时输入框仍可见。

验收：

- 打开 Thread 不会把聊天区挤成窄列。
- Thread 内消息、Markdown、表格可读。
- 关闭 Thread 后回到原频道滚动位置。

### 4. Composer 移动端适配

输入框是移动端最容易坏的区域。

要求：

- Composer 固定在底部或贴在可视区域底部。
- iOS/Android 软键盘弹出时，输入框不能被遮挡。
- 发送按钮不能被挤出屏幕。
- 文本域高度可随输入增长，但有最大高度。
- mention suggestion 在移动端不超出屏幕。
- Thread composer 和 channel composer 行为一致。

建议：

- 使用 `env(safe-area-inset-bottom)` 处理底部安全区。
- Composer 容器加 `padding-bottom: max(12px, env(safe-area-inset-bottom))`。
- 避免使用会在移动浏览器里表现不稳定的 100vh，优先使用 `100dvh` 并提供 fallback。

### 5. 消息与 Markdown 窄屏适配

要求：

- 消息内容 `min-width: 0`，允许换行。
- URL 使用 `overflow-wrap: anywhere`。
- 代码块横向滚动，不撑破页面。
- 表格放入横向滚动容器。
- 头像尺寸可略缩小，例如桌面 36px，移动 32px。
- Hover action 在移动端改为轻量按钮或长按/更多菜单，不依赖 hover。

验收样例：

- 北京穿衣建议表格在 375px 宽度下可横向滚动。
- 阿联酋长链接不会撑出屏幕。
- Thread 回复摘要不换行到不可读状态。

### 6. Touch Target 与可访问性

要求：

- 主要可点击元素触控区域至少 `40px`，推荐 `44px`。
- icon-only 按钮必须有 `aria-label`。
- Drawer 打开时焦点进入 drawer。
- Drawer 关闭后焦点回到触发按钮。
- ESC 关闭 overlay。
- 遮罩点击关闭，但面板内部点击不误关。

### 7. 横向溢出治理

全局要求：

- `body` 和主 app 不应出现水平滚动。
- 任何面板、消息、表格、代码块都不能撑破 viewport。
- 长文本统一处理：
  - `min-width: 0`
  - `overflow-wrap: anywhere`
  - 必要处 `text-overflow: ellipsis`

- mobile 下 right rail 变 overlay 或 full-screen sheet。

验收：

- 打开 AgentPanel / ThreadPanel 均不产生横向滚动。

### Task 6：消息内容窄屏治理

验收：

- 表格、代码块、长 URL 不撑破屏幕。
