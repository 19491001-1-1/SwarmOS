# v1.5.1 — Editable Agent Runtime

> **给 Claude Code / Codex 的执行指令：** 实现本文档描述的 v1.5.1。按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。完成后运行 `pnpm verify`；如果触及 Cloudflare Worker，也必须运行 `pnpm --filter @mini-slock/cloudflare exec wrangler deploy --dry-run`。如果触及 Web 构建，也运行 `VITE_API_BASE=https://xoxiang-hub.xingke0.workers.dev pnpm --filter @mini-slock/web build`。

## 背景

当前创建 Agent 时可以选择 `runtime`，但创建后 `PATCH /api/agents/:id` 不允许修改 runtime。实际使用中，用户经常会先随手创建一个 Agent，后面才发现它应该从 `claude` 改成 `codex` 或 `gemini`。

v1.5.1 是一个小版本，只补齐 Agent profile 的 runtime 可编辑能力，并保证 server、Cloudflare hub、Web 和 agent-facing CLI 行为一致。

## 目标

- 用户能在 Web Agent 详情页修改 Agent runtime。
- API 支持通过 `PATCH /api/agents/:id` 修改 runtime。
- Agent-facing CLI 支持修改 runtime，方便调试和自动化。
- 修改 runtime 时校验目标 machine 是否支持该 runtime。
- 修改运行中/工作中的 Agent 时行为可预期，不出现旧 runtime 继续被错误唤醒。
- Server 和 Cloudflare hub 行为一致。

## 非目标

- 不做 runtime 数据迁移，例如 Claude session 转 Codex session。
- 不自动转换 system prompt 或 model 名称。
- 不自动安装缺失 runtime。
- 不重做 Agent profile UI，只在现有详情/编辑入口补齐字段。
- 不改变 `RuntimeId` 的取值集合，仍使用 `claude | codex | gemini`。

## 行为规则

### Runtime 修改规则

允许修改：

- `inactive`
- `idle`
- `exited`
- `error`

谨慎处理：

- `starting`
- `running`
- `working`

对于 `starting/running/working`，第一版采用保守规则：

- 默认拒绝修改，返回 `409`。
- 错误信息说明需要先 stop Agent。
- 后续如果需要可增加 `force: true`，但 v1.5.1 不做。

### Machine 兼容性

修改 runtime 时必须校验：

- 如果 Agent 有 `machineId`，该 machine 必须存在且 `runtimes` 包含目标 runtime。
- 如果 Agent 没有 `machineId`，允许保存 runtime，但 start 时仍按现有逻辑选择支持该 runtime 的 connected machine。
- 如果指定 `machineId` 和 `runtime` 同时 patch，必须用 patch 后的 machine/runtime 组合校验。

### Side effects

修改 runtime 成功后：

- `model` 保持不变，除非请求同时传入新 `model`。
- `autoStart` 保持不变，但如果 Agent 当前不可安全修改则拒绝。
- `status` 不应被静默改写，除非现有 update flow 已有明确规则。
- 广播 `agent:update` / `agent:updated` 事件。
- 后续 start/deliver 必须使用新 runtime 的 runtime config。

## 数据模型

`Agent.runtime` 已存在，不新增字段。

扩展 patch request：

```ts
type PatchAgentRequest = {
  runtime?: RuntimeId;
  machineId?: string;
  displayName?: string;
  description?: string;
  model?: string;
  systemPrompt?: string;
  autoStart?: boolean;
  envVars?: Record<string, string>;
  organization?: AgentOrganization;
};
```

错误建议：

```json
{
  "error": "Cannot change runtime while agent is working. Stop the agent first."
}
```

```json
{
  "error": "Machine does not support runtime codex"
}
```

## 实现任务

### Task 1：Shared validation

在 `packages/shared/src/validation.ts` 中：

- `PatchAgentRequestSchema` 增加 `runtime: RuntimeIdSchema.optional()`。
- refine 的“至少一个字段”判断包含 `runtime`。

测试：`packages/shared/test/protocol.test.ts`

- `PATCH` payload 允许 `{ runtime: 'codex' }`。
- 非法 runtime 返回校验失败。
- 空 patch 仍失败。

### Task 2：Server API

在 `packages/server` 中补齐 runtime patch 行为：

- `PATCH /api/agents/:id` 接受 runtime。
- 修改前读取现有 Agent 和目标 machine。
- 对 `starting/running/working` 返回 `409`。
- 校验目标 machine 支持目标 runtime。
- 成功后持久化并广播 agent update。

测试：`packages/server/test/agentsApi.test.ts`

- 可以把 inactive Agent 从 `claude` 改为 `codex`。
- 运行中 Agent 修改 runtime 返回 `409`。
- 绑定 machine 不支持目标 runtime 时返回 `400`。
- 同时 patch `{ machineId, runtime }` 时用新组合校验。

### Task 3：Cloudflare parity

在 `packages/cloudflare/src/index.ts` 中实现同样规则：

- `PatchAgentRequestSchema` 已共享，Cloudflare route 需要接入 runtime。
- `updateAgent` 已能写 runtime 时，仍要在 route 层补齐状态和 machine runtime 校验。
- 保持错误码和 server 一致。

测试：`packages/cloudflare/test/hub.test.ts`

- 覆盖成功修改 runtime。
- 覆盖 busy 状态拒绝。
- 覆盖 machine runtime 不兼容。

### Task 4：Agent-facing CLI

在 `packages/daemon/src/agentCli.ts` 中增加或扩展命令：

```bash
xoxiang agent update <agentId> --runtime codex
```

如果已有 `agent update`，只补齐 `--runtime` 参数；如果没有，新增最小命令。

要求：

- 输出更新后的 Agent JSON。
- 透传 server 错误信息。
- CLI 文档同步更新 `docs/agent-facing-cli-reference.md`。

测试：`packages/daemon/test/agentCli.test.ts`

- `xoxiang agent update agent-1 --runtime gemini` 调用 `PATCH /internal/agent/...` 或公开 API 的既有更新入口。
- 非法 runtime 在 CLI 或 server schema 层失败。

### Task 5：Internal Agent API

如果当前 internal agent API 没有 Agent patch 能力，新增窄接口：

```http
PATCH /internal/agent/:agentId/agents/:targetAgentId
```

用途：

- Agent 可以在用户明确要求时调整另一个 Agent 的 runtime。
- 必须复用公开 API 的校验规则。

限制：

- 不允许 Agent 修改自己的 runtime 且保持当前进程继续执行；如果 target 是自己且当前状态 busy，应返回 `409`。
- 不引入权限系统，沿用 agent token 认证。

### Task 6：Web UI

在现有 Agent detail / edit panel 中加入 runtime selector：

- 展示当前 runtime。
- 可选项来自固定 `claude/codex/gemini`，后续可改为 machine capability 动态列表。
- 如果 Agent 已绑定 machine，禁用或标注该 machine 不支持的 runtime。
- 对 busy Agent 显示提示：需要 stop 后再改。
- 保存失败时展示后端错误。

测试：

- `packages/web/test/App.test.tsx` 或组件测试覆盖 runtime select 出现和 patch payload。
- 如有 i18n 文案，补齐中英文 key。
