# v1.8 Agent Memory / Skills / Runtime Roadmap

## 版本目标

让 agent 能跨任务积累经验：有 per-agent memory，成功任务可沉淀为 skill，Codex/Gemini 具备 session carryover，并把 runtime driver 变成可扩展 plugin。

## 非目标

- 不做大规模向量数据库依赖；语义 memory 放到 v1.9。
- 不自动把所有输出写入长期记忆；必须有过滤和 scope。
- 不要求所有 runtime 支持真 session resume；先做摘要 carryover。

## 为什么现在做

Crewden agent 已能读 inbox、搜 knowledge、委派任务，但 memory 仍偏全局；Codex/Gemini one-shot 冷启动会丢上下文；成功任务无法变成下次可复用的工作方式。

## 核心用户故事

1. 作为 coding agent，我能记住“这个 repo 测试命令是什么、常见坑是什么”。
2. 作为用户，我批准一个成功任务后，系统能提取可复用 skill 草稿。
3. 作为 Codex/Gemini runtime，我下一轮至少能看到上一轮摘要和未完成事项。
4. 作为开发者，我能新增 opencode/cursor-agent driver，而不用改 agentProcessManager 主逻辑。

## 数据模型/API/协议建议

- Knowledge 增加 `scope: global | workspace:<id> | agent:<id>`、`kind: note | skill | runbook | memory`。
- CLI：`crewden knowledge search --scope agent:self`、`crewden memory add --text ... --tags ...`。
- Workspace 文件：`.crewden/agents/<agentId>/session-summary.md`。
- Runtime plugin interface：`RuntimeDriverFactory`, `RuntimeDriverManifest`, dynamic config loading。

## TDD 任务拆解

### Task 1：knowledge scope 支持 agent memory

- Failing test：`filters knowledge by agent scope` in `knowledgeApi.test.ts`。
- 命令：`pnpm --filter @crewden/server test -- knowledgeApi.test.ts -t "agent scope"`
- 期望失败：search 不区分 agent scope。
- 最小实现：knowledge schema 增 scope；route search/create 解析 scope；默认 global。
- 通过标准：agent A 搜不到 agent B private memory，global 仍可见。
- Commit：`feat(knowledge): add scoped agent memory`

### Task 2：agent CLI memory 命令

- Failing test：`crewden memory add sends scoped knowledge request` in `agentCli.test.ts`。
- 命令：`pnpm --filter @crewden/daemon test -- agentCli.test.ts -t "memory add"`
- 期望失败：命令不存在。
- 最小实现：在 `agentCli.ts` 增 `memory add/search/list`，映射到 knowledge API，默认 `agent:<currentAgentId>`。
- 通过标准：CLI 生成正确 request；缺 agent id 时清晰报错。
- Commit：`feat(cli): add per-agent memory commands`

### Task 3：成功任务 skill capture 草稿

- Failing test：`creates skill draft after approved task completion`。
- 命令：`pnpm --filter @crewden/server test -- tasksApi.test.ts -t "skill draft"`
- 期望失败：done/review approved 不触发 skill。
- 最小实现：在 task done + review approved 后提取 acceptanceCriteria、commands/evidence、handoffNotes，生成 `kind: skill` knowledge draft，状态 `draft`。
- 通过标准：只生成草稿，不自动启用；敏感词被 redacted。
- Commit：`feat(skills): capture successful task drafts`

### Task 4：Codex/Gemini session carryover 摘要

- Failing test：`injects previous session summary into codex prompt` in `drivers.test.ts`。
- 命令：`pnpm --filter @crewden/daemon test -- drivers.test.ts -t "session summary"`
- 期望失败：driver prompt 不读 summary。
- 最小实现：`agentWorkspace.ts` 提供 read/write summary；`codex.ts/gemini.ts` 在 prompt 前注入摘要；close 后写最新简短摘要。
- 通过标准：summary 大小有限制；不存在时无影响。
- Commit：`feat(runtime): add session carryover summaries`

### Task 5：Runtime plugin interface

- Failing test：`loads runtime driver from plugin manifest`。
- 命令：`pnpm --filter @crewden/daemon test -- drivers.test.ts -t "plugin manifest"`
- 期望失败：driver registry 是静态。
- 最小实现：`drivers/types.ts` 定义 manifest/factory；`agentProcessManager.ts` 使用 registry；内置 Claude/Codex/Gemini 迁移到 registry。
- 通过标准：旧配置不变；测试 fake plugin 可加载。
- Commit：`feat(runtime): introduce driver plugin registry`

## 测试用例清单

- Happy：agent memory add/search；approved task 生成 skill draft；Codex prompt 带摘要；fake runtime plugin 执行。
- Edge：scope 缺省；summary 超长截断；skill capture 无 evidence；plugin 名冲突。
- Failure：非法 scope；memory 写入失败；summary 文件损坏；plugin load throw。
- Concurrency/restart：两个 agent 同时写 memory；daemon restart 后 summary 仍在；skill capture 重试不重复生成。

## 验收标准

- Memory 有 scope 隔离，默认不泄露 agent private memory。
- Skill capture 只产草稿，用户/agent 可 review。
- Codex/Gemini carryover 不依赖真实 CLI 也能测试。
- Runtime plugin 不破坏现有 Claude/Codex/Gemini 行为。

## 风险与降级

- Memory 污染：限制写入长度、tag、scope；默认 private。
- Skill 自动生成质量不稳：先 draft，不自动加载。
- Plugin 动态加载风险：只允许配置白名单路径。

## 版本依赖

依赖 v1.6 audit/状态可靠性；可与 v1.7 并行部分开发，但 skill capture 最好读取 v1.7 verification evidence。
