# v1.2 — Chat-Native Goal Alignment

> **给 Claude Code / Codex 的执行指令：** 实现本文档描述的 v1.2。按 TDD 工作：先写失败测试，再实现，再跑测试确认通过。完成后运行 `pnpm verify`；如果触及 Cloudflare Worker，也必须运行 `pnpm --filter @crewden/cloudflare exec wrangler deploy --dry-run`。如果触及 Web 构建，也运行 `VITE_API_BASE=https://crewden-hub.xingke0.workers.dev pnpm --filter @crewden/web build`。

## 背景

v1.1 解决了 Goal Brief 和任务拆解的数据结构。v1.2 解决用户体验：用户不应该离开聊天去填表。系统应在当前 channel/thread 中完成目标识别、澄清、计划预览、参与 Agent 推荐和用户确认。

## 目标

- 在聊天中识别目标型请求。
- 对信息不足的目标生成澄清问题。
- 在 Thread 中维护目标对齐过程，避免污染主频道。
- 生成计划预览，说明推荐哪些 Agent 以及原因。
- 用户确认后创建 Goal Brief 和任务。
- 高风险或不确定目标默认等待用户确认。

## 非目标

- 不做独立 Boss Dashboard。
- 不做复杂审批流。
- 不做自动执行所有任务，v1.3 处理。
- 不替代用户确认高风险计划。

## 核心概念

```ts
type GoalAlignmentStatus =
  | 'needs_clarification'
  | 'awaiting_confirmation'
  | 'confirmed'
  | 'cancelled';

type GoalAlignment = {
  id: string;
  channelId: string;
  threadRootId: string;
  sourceMessageId: string;
  goalId?: string;
  status: GoalAlignmentStatus;
  questions: string[];
  answers: string[];
  planSummary?: string;
  recommendedAgentIds: string[];
  recommendationReasons: Record<string, string>;
  riskLevel: 'low' | 'medium' | 'high';
  createdAt: string;
  updatedAt: string;
};
```

## 实现任务

### Task 1：Goal intake classification

在 shared/hub-core 中实现轻量规则函数：

```ts
classifyMessageIntent(message: Message): 'chat' | 'task' | 'goal';
```

最低规则：

- 包含“帮我做/实现/调研/规划/推进/完成”等动作词，且目标需要多步协作时判为 `goal`。
- 明确单步事项可判为 `task`。
- 普通对话为 `chat`。

注意：这是系统辅助，不需要假装完美。允许用户手动 `→ Goal` 覆盖。

测试：

- `packages/hub-core/test/goalAlignment.test.ts`

### Task 2：Alignment API

Server 和 Cloudflare 增加：

```http
POST  /api/messages/:id/start-goal-alignment
GET   /api/goal-alignments/:id
PATCH /api/goal-alignments/:id
POST  /api/goal-alignments/:id/confirm
POST  /api/goal-alignments/:id/cancel
```

`confirm` 做两件事：

1. 创建或更新 `GoalBrief`。
2. 创建任务草案对应的 `Task`。

测试：

- `packages/server/test/goalAlignmentApi.test.ts`
- `packages/cloudflare/test/hub.test.ts`

### Task 3：Agent recommendation

基于 v1.0 的 role/capability resolve：

- 推荐 owner Agent。
- 推荐 reviewer Agent。
- 给出每个推荐的原因。
- 如果找不到合适 Agent，返回缺口说明，而不是随便选。

建议复用：

```ts
resolveAgent(query)
```

新增纯函数测试推荐逻辑。

### Task 4：Web chat flow

在 `ChannelView` 和 Thread 中实现：

- 目标型消息显示轻量 action：`Plan goal`。
- 点击后打开 Thread 或右侧 alignment panel。
- Panel 展示：
  - objective draft
  - clarification questions
  - success criteria
  - task plan preview
  - recommended agents + reasons
  - `Confirm plan` / `Revise` / `Cancel`
- 计划确认后主频道显示一条结构化 summary message，包含 Goal 和任务链接。

要求：

- 不要用 marketing hero 或大卡片。
- UI 保持工作台风格，密集但清晰。
- 计划预览必须能在小屏幕查看。

### Task 5：Agent prompt & CLI guidance

更新 `simpleToolBridge`：

- Agent 遇到目标型请求，不要直接把所有工作转派出去。
- 先用 goal/alignment 上下文判断是否需要澄清。
- 计划要说明“为什么找这些 Agent”。
- 风险高时等待用户确认。

CLI 增加：

```bash
crewden goal align <messageId>
crewden goal alignment read <alignmentId>
crewden goal alignment confirm <alignmentId>
```

## Implementation Notes

- `packages/hub-core/src/goalAlignment.ts` contains lightweight message intent classification, clarification question generation, risk inference, and role/capability-based agent recommendation.
- Server and Cloudflare expose matching public alignment APIs and internal agent-facing endpoints.
- Web `Plan goal` now opens a chat-native alignment panel with questions, plan preview, recommended agents, and `Confirm plan`.
- Agent CLI supports `crewden goal align`, `crewden goal alignment read`, and `crewden goal alignment confirm`.

## 手动验证

1. 发送“帮我做一个 Mac 全局语音输入法，从需求到技术方案都安排一下”。
2. 点击 `Plan goal`。
3. 确认系统在 Thread 中追问或生成计划。
4. 检查推荐 Agent 是否包含产品/工程/测试，且说明原因。
5. 点击确认，确认 Goal 和任务被创建。
6. 任务详情里能看到 alignment 来源。
