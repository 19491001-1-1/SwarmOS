# v0.9.1 — UI Polish: Adaptive Right Rail & Sidebar Hover

> 本文档是 v0.9 后的界面微调版本规划，可直接交给 Claude Code / Codex 执行。执行时从 `main` 新建 `feat/v0.9.1-ui-polish` 分支，按 TDD/截图验收推进，完成后运行 `pnpm verify`。

## 目标

v0.9.1 不新增大功能，专注把当前工作台的布局和交互打磨得更接近成熟协作工具：

1. 最右侧 Agent 边栏不再常驻，占用主聊天阅读空间。
2. 右侧区域改成按需打开的辅助面板：Agent、Thread、详情等互斥展示。
3. 左侧导航和菜单增加清晰但克制的 hover/active 动效。
4. 参考截图里的交互感：左侧菜单项 hover 后有高亮、边框、轻微位移或阴影，但整体仍保持当前产品的像素风识别度。

## 背景

当前页面有三栏常驻：

- 左侧工作区/频道/Agent 列表
- 中间聊天
- 右侧 Agent 面板

问题：

- 右侧 Agent 面板长期占宽，但多数时候用户并不需要一直看它。
- v0.9 引入 Thread 后，如果右侧 Agent 面板仍常驻，Thread 会进一步挤压聊天区。
- 左侧菜单现在状态比较静态，hover 缺少反馈，用户不容易感知哪些区域可点击。

v0.9.1 先解决这些 UI 结构问题，为 v0.9 Thread 和 Markdown 长内容阅读让路。

## 非目标

- 不实现 v0.9 的 Markdown/Thread 数据模型。
- 不重做整体视觉品牌。
- 不新增复杂设置页。
- 不改变现有 API。
- 不做移动端完整重构，但不能让窄屏明显恶化。

## 产品要求

### 1. 右侧边栏不再常驻

默认布局：

- 左侧 sidebar 常驻。
- 中间主内容占据剩余宽度。
- 右侧 AgentPanel 默认收起。

打开方式：

- 顶部或左侧提供“成员/Agent”入口。
- 点击 Agent 入口后，右侧以 drawer/side panel 形式打开。
- 再次点击或点击关闭按钮后收起。
- ESC 可关闭。

与 Thread 的关系：

- v0.9 Thread 面板打开时，右侧优先展示 Thread。
- AgentPanel 和 ThreadPanel 不应同时常驻挤压主聊天。
- 若 AgentPanel 已打开，再打开 Thread，应切换右侧面板内容为 Thread。
- 若 Thread 关闭，可以回到之前打开的 AgentPanel，也可以直接收起；实现时选择更简单稳定的方式即可，但行为要一致。

尺寸建议：

- Desktop panel width：`320px` 到 `380px`。
- 大屏最大不超过 `420px`。
- 小屏使用 overlay drawer，不挤压主内容。
- 主聊天区域最小宽度应保持可阅读，建议不低于 `720px`，低于时右侧改 overlay。

### 2. 左侧菜单 hover/active 动效

范围：

- Search
- Threads
- Tasks
- Saved
- Channels 下的 channel item
- Direct Messages 下的 DM item
- Workspace selector
- 顶部 tab icon
- 新建 channel / new agent 等 icon button

Hover 要求：

- 鼠标移上可点击项时有明确反馈。
- 反馈可以包括：
  - 背景色变浅或变亮
  - 左侧/外圈边框强调
  - 轻微 `translateX(2px)` 或 `translateY(-1px)`
  - 像素风 shadow，例如 `2px 2px 0 #000`
  - icon 或文字颜色加深
- 过渡时间建议 `100ms - 160ms`。
- 动效不能引发布局跳动：预留边框宽度或使用 outline/box-shadow。
- active 状态必须比 hover 更明确。
- disabled / empty 状态不能误导为可点击。

Active 状态：

- 当前选中的频道、Threads、Tasks 等保留稳定高亮。
- active item 可使用粉色背景或当前设计中的强调色。
- active hover 不应覆盖掉 active 语义。

Accessibility：

- `:focus-visible` 要有键盘可见焦点。
- 可点击项使用 button/a 或带 role 的元素。
- hover 反馈也要在 keyboard focus 下出现。

### 3. 左侧信息结构微调

参考截图，左侧可以更像协作工具：

- 顶部 workspace selector 保留。
- 主导航项独立分组：Search / Threads / Tasks / Saved。
- Channels 和 Direct Messages 分组标题可折叠。
- Count badge 要低调，不抢主视觉。
- DM 在线状态点放右侧，大小固定。

不要求一次完成全部结构调整，但 v0.9.1 至少要做到：

- 现有 Sidebar 的 hover/active/focus 明显改善。
- 菜单项高度、间距、点击区域一致。
- 长 channel/agent 名称截断清晰。

### 4. 视觉约束

- 保留当前黄色品牌侧栏，但减少过重边框对主内容的干扰。
- 不新增渐变球、装饰 blob。
- 不使用过大的圆角；现有像素风可保留，但 hover 动效要更顺滑。
- 主内容区域背景保持安静。
- 所有文字不能溢出按钮或菜单项。

### Task 4：微交互动效

建议 CSS：

```css
.sidebarItem {
  transform: translateX(0);
  transition: background-color 140ms ease, transform 140ms ease, box-shadow 140ms ease;
}

.sidebarItem:hover,
.sidebarItem:focus-visible {
  transform: translateX(2px);
  box-shadow: 2px 2px 0 #000;
}

.sidebarItemActive {
  background: #ff4d8d;
  box-shadow: 3px 3px 0 #000;
}
```

要求：

- 实际实现可以用 inline style 或 CSS module，但不要复制导致样式碎片化。
- 如果当前项目没有 CSS module，可先用集中常量或单文件 CSS。
